## Lean 4 Metaprogramming Guidelines

### Basic Structure
A tactic in Lean 4 is defined using the `Lean.Elab.Tactic` framework:

```lean
import Lean

open Lean Elab Tactic Meta

elab "my_tactic" : tactic => do
  -- tactic implementation
  return ()
```

### Key Metaprogramming Concepts

1. **Accessing the Goal**: Use `getMainGoal` to get the current goal
2. **Getting Goal Type**: Use `getMVarType` to get the type of a metavariable
3. **Closing Goals**: Use `closeMainGoal` or `assignExprMVar`
4. **Creating Expressions**: Use `mkAppM`, `mkConst`, etc.
5. **Running Other Tactics**: Use `evalTactic` with syntax quotations

### Common Patterns

```lean
-- Get the main goal and its type
elab "inspect_goal" : tactic => do
  let goal ← getMainGoal
  let goalType ← getMVarType goal
  logInfo m!"Goal type: {goalType}"

-- Apply a lemma
elab "apply_lemma" : tactic => do
  let goal ← getMainGoal
  let lemmaExpr ← mkConst ``MyLemma
  let newGoals ← apply goal lemmaExpr
  replaceMainGoal newGoals

-- Run another tactic
elab "my_wrapper" : tactic => do
  evalTactic (← `(tactic| simp))
```

### Working with Expressions

```lean
-- Check if expression matches a pattern
def isAddition (e : Expr) : Bool :=
  e.isAppOfArity ``HAdd.hAdd 6

-- Decompose an application
def getAppArgs (e : Expr) : Array Expr :=
  e.getAppArgs
```
