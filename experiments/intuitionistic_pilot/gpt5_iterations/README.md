# GPT-5 Iterations — Intuitionistic Implication Tactic

This directory contains 13 Lean 4 files generated by GPT-5 (OpenAI) during the
pilot experiment. They document the **iterative error-correction approach** to
tactic generation: generate code, feed compiler errors back, repeat.

The tactic target: prove goals of the form `P₁ → P₂ → … → Pₙ → Pᵢ`
(implication fragments of intuitionistic propositional logic where the conclusion
is one of the hypotheses).

---

## Iteration History

### `PropositionalLogic1/` — First session (8 files)

Iterative development starting from scratch. Files are numbered to show the sequence:

| File | Description |
|------|-------------|
| `propositions_1.lean` | First attempt: hardcoded for exactly 2-premise goals (`A → B → A`, `A → B → B`). Works for those cases but does not generalise. Comment in file: "matches the form of the statements too closely." |
| `propositions_2.lean` | Second attempt: tries to handle n premises by counting `→` nesting. Still brittle. |
| `propositions_3_1.lean` — `propositions_3_5.lean` | Five repair rounds on the 3rd generation attempt. Each round feeds compiler errors back. Shows the non-convergence issue: errors are fixed but new ones appear. |
| `propositions_4_1.lean` | Fourth generation attempt after spec clarification. More structured but still fails on edge cases. |

### `PropositionalLogic2/` — Second session (3 files)

Approach refined: use `whnf` + `forallE` matching to peel off implications.

| File | Description |
|------|-------------|
| `propositions_1.lean` | First attempt of session 2: `collectPremises` with `whnf` + pattern matching |
| `propositions_2.lean` | Repair after type errors |
| `propositions_3.lean` | Working version for the basic class, gas-free |

### `PropositionalLogic3/` — Third session — spec-first approach (2 files)

After switching to spec-first: the LLM was first asked to characterise the formula
class mathematically, then to implement the tactic.

| File | Description |
|------|-------------|
| `propositions_1.lean` | Implementation guided by the spec. Uses mutual recursion (`buildProof` / `tryFuncs` / `buildArgs`) with a gas counter for termination. |
| `propositions_1_2.lean` | Repaired version: fixes `FVarSet` import and minor elaboration issues. **Final working implementation.** |

---

## Key Observation

The 8 files in `PropositionalLogic1/` illustrate the failure mode of the iterative
approach: the model corrects local syntax errors but lacks a semantic anchor, so
it cannot verify whether the tactic is actually correct.

The 2 files in `PropositionalLogic3/` illustrate the spec-first approach: starting
from a mathematical characterisation (`buildProof` as a backward proof search),
the implementation converges in two steps.

This observation motivated the spec-first pipeline used in the `limit_auto` study.
See `experiments/intuitionistic_pilot/README.md` for the full pilot write-up.
