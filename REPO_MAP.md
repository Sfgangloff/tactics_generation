# Repository Map

This file describes the structure of the repository and marks what is
relevant for the paper.

**Paper**: "Generating Lean 4 Tactics from User Specifications: An Empirical
Study of LLM-Assisted Metaprogramming Strategies" (ITP short paper)

---

## Paper-critical material

```
paper/
  main.tex                       ← paper draft (LaTeX)
  paper_plan.md                  ← section outline and evidence table

experiments/
  intuitionistic_pilot/          ← Section 3: pilot experiment
    README.md
    prove_impl_tautology.lean    ← iterative approach: first tactic attempt
    prove_impl_tautology.spec.md
    prove_paren_impl_taut.lean   ← iterative approach: refined version
    prove_paren_impl_taut.spec.md
    identity_intro.lean          ← spec-first approach: identity variant
    identity_intro.spec.md
    gpt5_iterations/             ← the 13 GPT-5 generated iterations
      PropositionalLogic1/       ← 8 files: iterative error-correction approach
      PropositionalLogic2/       ← 3 files: intermediate versions
      PropositionalLogic3/       ← 2 files: final spec-first version

  limit_auto/                    ← Section 4: 2×2 study (main contribution)
    spec.md                      ← shared specification for all 4 conditions
    README.md                    ← study design explanation
    one_run/
      limit_auto.one_run.lean    ← Condition A: raw one-shot (no plan, no LSP)
    plan_without_lsp/            ← Condition B: plan, no LSP
      plan.md
      milestone1.lean … milestone5.lean
    one_shot_with_lsp/           ← Condition C: no plan, with LSP
      output.lean
    plan_with_lsp/               ← Condition D: plan + LSP (best result)
      plan.md
      lsp_log.md                 ← full record of LSP tool calls
      milestone1.lean … milestone9.lean

  other_tactics/                 ← Section 7 (future work): other user specs
    decide_list_theory/          ← second tactic (LSP-guided, 5 milestones)
    specs/                       ← 13 specs generated by the pipeline
```
